Convention Over Configuration

  1. Convention for configuration

  2. All micro configuration files are to be placed under the source/config folder with a .erb extension suffix after their real extension. This will allow expansion to occur.

  3. Test Naming Convention

  4. All tests for a component X will be in a file name XSpecs.cs. And that file will contain an outer class with the same name as the file, which all contexts will live under.
